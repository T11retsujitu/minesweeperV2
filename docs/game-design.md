## 1. コンセプトと核仮説

**マインスイーパー × ダンジョンRPGの戦闘ローグライト。**
数字を読む推理で地雷の位置を特定し、その地雷を「敵への攻撃資源」として遠隔起爆して敵を倒す。

- **核仮説(検証中)**: 数字を読んで特定した地雷を、敵への攻撃として起爆する行為そのものが気持ちよいか
- **差別化点**: 地雷=脅威ではなく攻撃資源。「開けて避ける」ゲームではなく「特定して使う」ゲーム
- **北極星**: PVビジョン(docs/pv-vision-roadmap.md §1)—「盤上に主人公がいて、周囲にタイルが開き、数字と敵が見える」1カットで伝わる構図

## 2. 現行ルール(ruleset = "phase2_avatar"、UIデフォルト)

### 2.1 盤面と初期状態

| 項目 | 値 | 備考 |
|------|-----|------|
| 盤面 | 7×7、地雷9個 | `BOARD_W/H`, `MINE_COUNT` |
| 敵 | 1体、安全セル上に固定 | HP6・攻撃2・カウントダウン3 |
| プレイヤー | HP10、**盤上に位置を持つ** | 初期位置: fixture=(1,3) / ランダム=敵ゾーン外の安全セル(開放して出現) |
| 固定盤面 | `phase1_core_demo`(初期開放8マス) | 座標変更禁止(ソルバー検証済みオラクル) |
| ランダム盤面 | シード決定的。敵は内側5×5、敵周囲3×3に地雷ちょうど3個 | 初手安全移設あり(D10) |

### 2.2 プレイヤーの行動

| 行動 | ターン消費 | 条件・効果 |
|------|-----------|-----------|
| **開放(REVEAL)** | 1 | **自分の8近傍**の未開放セルのみ。0セルならフラッド連鎖(連鎖範囲は無制限)。地雷なら誤爆 |
| **移動(MOVE)** | 1 | **8近傍**の開放済み・非クレーター・非敵セルへ。開放済みセルのタップ=移動 |
| **起爆(DETONATE)** | 1 | **盤面のどこでも**フラグ済みセルを起爆可(遠隔=核仮説のメカニクス)。プレビュー→確認の2段階 |
| フラグ設置/解除 | 0 | 任意セル。長押し or 右クリック |

- 合法手はUIで常時ハイライト(移動先=青緑 / 開放可能=黄)
- 非合法手(遠すぎる開放・不正な移動先)はターン非消費で拒否され、理由がステータスに出る

### 2.3 ダメージルール

| 事象 | ダメージ | 対象 |
|------|---------|------|
| 起爆の爆発 | 爆心4 / 隣接2(Chebyshev半径1の3×3) | 範囲内の敵 **とプレイヤー**(スプラッシュ、下記) |
| **起爆スプラッシュ** | 実質最大2(プレイヤーは地雷セル=爆心に立てない) | 意図起爆のみ。**位置取りのリスク**: 起爆前に爆風圏から離れるかの判断が生まれる |
| 誤爆(地雷を開放) | プレイヤーに**フラット3**(爆風加算なし、D24)。爆風は範囲内の敵に通常適用 | 誤爆セルはクレーター化(歩行不可) |
| dud(安全セルを起爆) | 0。当該セルが**遠隔開放**される | 肩透かしだが、移動封鎖時の脱出弁として機能(D26) |
| 敵の攻撃 | 2 | カウントダウン0で発動、カウントは3に戻る |

### 2.4 ターン解決順序(§5.8準拠・不変)

1. 入力確定 → 2. 開放/起爆/移動を解決 → 3. 爆発ダメージ(敵→プレイヤーspalshの順)→ 4. 敵死亡 → 5. **勝利判定(成立なら以降スキップ=勝利優先)** → 6. カウントダウン減 → 7. 0なら敵攻撃 → 8. プレイヤー死亡 → 9. 敗北判定 → 10. UI同期

- **移動もカウントダウンを進める**(D25)= ターン圧力。1手の価値が「情報を取る/位置を直す/攻撃する」の三択になる
- 勝利: 敵HP0 / 敗北: プレイヤーHP0(同時成立は勝利優先、D6)

### 2.5 詰みが存在しないことの保証

勝利条件は敵撃破であり全盤面開放は不要。移動経路がクレーター等で完全封鎖されても、フラグ+起爆(dud)による遠隔開放で情報取得が続行できる(D26)。敗北はHP0のみ。

### 2.6 固定盤面の攻略オラクル(tests/test_avatar_walkthrough.gd で全手固定)

- **勝利ライン(7ターン、最終HP6)**: 起爆(1,1)→起爆(0,1)→開放(1,4)[被弾-2]→開放(2,4)→**移動(2,4)**→開放(1,5)[11マスフラッド、被弾-2]→起爆(0,3)で勝利
- **敗北ライン(5ターン)**: 誤爆(0,3)→移動(2,2)→誤爆(1,1)[被弾-2]→移動(3,2)→誤爆(4,1)で敗北
- 設計意図: 勝利には推理(部分集合推理を含む)と移動による経路選択の両方が必要。誤爆3回+敵攻撃1回でちょうど敗北ラインに乗る

## 3. 旧ルール(ruleset = "phase1"、テスト互換用に保存)

## 4. 体験レイヤ(ジュース・パス、pv-vision-roadmap Step 1 完了)

ルールではないが体験の一部として記録。すべてイベント購読のみ(domain非依存)で実装:

- **因果の可視化**: 敵バッジにカウントダウン数字(残1で「1!」点滅)→ 攻撃時は敵セル発光→弾がアバターへ飛翔→着弾でフロート+HPバー減少。「なぜHPが減ったか」が1本の演出で繋がる(課題1対応)
- **ダメージの帰属表示**: -N(与ダメ、敵セルから)/ -N MINE!(誤爆、誤爆セルから)/ -N ENEMY ATK / -N SPLASH!(アバター位置から)
- **起爆の手応え**: 爆心フラッシュ+パーティクル+ヒットストップ+時間差リング+画面シェイク(誤爆1.5倍)。dudは弱演出で「不発の肩透かし」を体感区別
- HPバー(減少痕ゴースト付き)、勝敗オーバーレイのフェード/スケール演出
- アセットは全て描画プリミティブの仮置き(画像・音は生成AIで後日差し替え予定)

### ルール変更の意図(Phase 1 → アバター)

1. **空間の意味付け**: 盤面が「情報の表」から「自分が立つ場所」になり、敵との距離・爆風との距離が実感を持つ
2. **1手の重み**: 移動がターンを消費するため「どこから情報を取るか」の経路計画が推理に加わる
3. **リスクの対価**: 遠隔起爆は無傷の攻撃だったが、スプラッシュ導入で「安全圏に下がってから起爆する」位置取りの駆け引きが生まれる
4. **核仮説は不変**: 「数字を読んで地雷を特定し敵に向けて起爆する」ループ自体には手を加えていない(遠隔起爆維持=R5)
