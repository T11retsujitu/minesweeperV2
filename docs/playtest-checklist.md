## 固定盤面の想定ライン(参考: ネタバレ)

> **注意**: 以下は Phase 1(盤外プレイヤー)ルール当時の記述。現行アバタールールの攻略オラクル(勝利7T/敗北5T/アグロ線4T、縄張り半径1準拠)は docs/game-design.md §2.6 が正本。

設計上の最短論理ライン(6ターン、被弾1回・残HP8で勝利):

1. (2,2)=1 の未開放隣接は (1,1) のみ → フラグ→起爆(敵6→4)
2. (1,2)=3 と (1,3)=1 の組合せで (0,1) が確定 → 起爆(敵4→2)
3. 同じ推理で (0,4),(1,4),(2,4) は安全 → (1,4) 開放(ここで敵攻撃、HP10→8)
4. (2,4) 開放 → (2,3)=1 から (3,4) が地雷と確定(ただし敵射程外=使う価値なし、が分かる)
5. (2,4)=1 から (1,5) 安全 → 開放すると 0 で左下が一気に開く
6. (0,4)=1 から (0,3) 確定 → 起爆(敵2→0、勝利)

深い推理を省いた場合は (0,2) vs (0,3) の五分五分ギャンブルとしても遊べる(dud なら1ターン損で情報確定)。

## 発見された課題

### 課題1: 敗北条件・被ダメージの原因が分かりにくい(2026-07-16 初回プレイテスト)

**症状**: マインスイーパーにHPがあるため「Enemyと遭遇したのかどうか」が分かりにくく、「HPがなぜ減ったのか?」となる。Defeat の条件が体感として伝わりにくい。

**分析**: 被ダメージ源は2系統(誤爆 = 自分の開放ミスで -3 / 敵攻撃 = カウント0で -2)だが、

- プレイヤーが盤面上に存在しない(handoff §4 `player_exists_on_board: false`)ため、「敵と遭遇する」という空間的な手掛かりがそもそも無い。HPの減少が抽象的なタイマーイベントとして起こる
- 敵攻撃の瞬間の演出が弱い(ステータスラベル1行+ログ+HPラベルの短い点滅のみ)。カウントダウン→攻撃の因果が画面の別々の場所に散っていて、体感で結びつかない

**対応候補**:

1. 攻撃演出の強化 — 攻撃瞬間の画面フラッシュ/シェイク、プレイヤーHPへのフローティングダメージ(「-2」)、短い停止
2. 予告の強化 — 敵セルの近くにカウントダウンを直接表示し、カウント1で「!」等の切迫表示
3. ダメージ内訳の明示 — HP減少時に理由を盤面上に大きく表示(「敵の攻撃 -2」/「誤爆 -3」)
4. (Phase 2 以降の設計課題)プレイヤーの盤面上への配置、または HP を別の資源(時間・シールド等)として再解釈するかの検討

**状態**: **解消済み(2026-07-16)** — 候補1〜3 はジュース・パス(Step 1)で実装しユーザー確認済み。候補4 も Phase 2 盤上アバターとして実装済み。

### 課題2: 敵の遠距離必中攻撃と「敵の隣が開けない」ストレス(2026-07-16 二回目プレイテスト)

**症状(ユーザー所感)**: ①遠距離攻撃が残っていてストレス ②攻撃の範囲も条件も分からない ③敵の隣が開放すべき非地雷マスのときイライラする ④敵に近づいても倒せず地雷も除去できない。

**分析**: 当時のカウントダウンはプレイヤー位置と無関係に毎ターン進む必中攻撃で、攻撃圏の描画もなかった(①②)。開放は対象セルの8近傍に立つ必要があり、敵の隣を開けるには敵の近くに行くしかない(③)。攻撃動詞が遠隔起爆のみだった(④)。

**対応(実装済み)**:

1. **縄張りルール+色塗り**(D31): 敵周囲を色塗りし、圏内にいるときだけカウントダウンが進む(圏外は「paused」表示)→ ①②解消
2. **縄張り半径 2→1 改訂**(D31改訂): 半径2(5×5)は 7×7 盤面の過半を占め「どこでも攻撃される」体感が残ったため、実プレイ判断で 3×3 に縮小。敵の隣の安全セルは距離2(圏外)から無リスクで開放可能に → ③解消
3. **bump攻撃+地雷除去**(D32/D33): 隣接した敵を殴れる・隣接地雷を安全に換金できる → ④解消

**状態**: 実装・検証済み。半径1の時間圧力(圏外で長考し放題になる副作用)は次回プレイテストの評価項目。
